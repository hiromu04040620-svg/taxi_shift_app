# 画面・機能仕様

各画面の入出力と挙動を定義する。

## 画面遷移図

```
[起動]
  ├─ (初回) → [オンボーディング] → [メイン]
  └─ (2回目以降) → [メイン]

[メイン (BottomNav)]
  ├─ [カレンダー]
  │    ├─ 日付タップ → [日詳細]
  │    │    └─ 編集 → [シフト編集]
  │    │    └─ 売上入力 → [売上入力]
  │    └─ 長押し → [種別クイック変更]
  ├─ [サマリー]
  │    └─ 月選択 → [月次詳細]
  └─ [設定]
       ├─ [サイクル管理]
       │    └─ [サイクル編集]
       ├─ [賃金設定]
       ├─ [通知設定]
       ├─ [データ管理] (エクスポート/インポート/バックアップ)
       └─ [このアプリについて]
```

## オンボーディング

初回起動時の3〜4ステップウィザード。

**Step 1: 勤務形態選択**
- ラジオ: 隔日勤務 / 日勤 / 夜勤
- 「あとで変更できます」の注記

**Step 2: サイクルプリセット選択**
- プリセット5種から選択（DOMAIN参照）
- 「カスタムで作る」も選択可
- カスタム選択時は配列ピッカーで種別を順に並べる

**Step 3: 起点日設定**
- 「次の出番はいつですか？」と質問
- カレンダーから選択
- 選択日を `workDay` 種別の起点として `startDate` に設定（cycle配列の workDay インデックスを考慮）

**Step 4: 月締め日**
- デフォルト15日。プルダウンで1〜31から選択
- 「給与明細の対象期間に合わせて設定してください」と説明

完了時 → メインのカレンダー画面へ

## カレンダー画面

### レイアウト

- 上部: 年月表示＋月切り替えボタン
- メイン: `table_calendar` を使った月カレンダー
- 各セル: 日付＋シフト種別の略号（出/明/公等）を色付き表示
- 下部: 選択日の概要カード（種別、売上、メモ）

### 振る舞い

- 日付タップ: 下部カードに該当日の情報表示
- 日付ダブルタップ or カードタップ: 日詳細画面へ遷移
- 日付長押し: モーダルで種別クイック変更（Override作成）
- スワイプ: 前月/翌月
- 上部の年月タップ: 月選択ピッカー

### 表示ロジック

```
1. 現在表示中の月の全日付を取得
2. 各日付について resolveShiftType(date, activePattern, overrides) で種別決定
3. セルに色付け＋略号表示
4. その月の Revenue がある日には小さい「¥」マーク
5. 改善基準告示警告がある月は上部にバッジ
```

### FAB（フローティングアクション）

- ＋ボタン: 「今日の売上を記録」のショートカット
- 当日が `workDay` でない場合は無効化（または警告表示）

## 日詳細画面

選択日の全情報を表示する画面。

**表示内容**:
- 日付（曜日表示込み）
- シフト種別（タップで変更モーダル）
- 売上情報（あれば。なければ「記録する」ボタン）
- 勤務時間情報
- メモ
- Override状態（「サイクルからの変更あり」表示、「元に戻す」ボタン）

**アクション**:
- 種別変更 → Override作成/更新
- 売上記録 → 売上入力画面
- 勤務時間記録 → 勤務時間入力モーダル
- 出番変更 → ペア日付選択モーダル

## シフト編集モーダル

任意の日付の種別を変更する。

**入力項目**:
- 種別（ラジオまたはチップ選択）
- ステータス（確定 / 希望中）
- 理由メモ（任意）

**保存時**:
- Override テーブルに upsert
- 隔日勤務で出番→公休に変更する場合、翌日の明け番を自動的に公休に変更するかをダイアログ確認

## 売上入力画面

出番日の売上を記録する。

**入力項目（必須）**:
- 総営収（数値入力、3桁カンマ自動）
- 走行距離（km、小数点1桁）
- 実車距離（km、小数点1桁）
- 乗車回数（整数）

**入力項目（任意）**:
- 税抜営収（総営収から自動計算もできるが、手動上書き可）
- 現金売上
- キャッシュレス売上
- その他売上
- 燃料補給額
- 写真添付（カメラ起動 or ギャラリー）
- メモ

**バリデーション**:
- 総営収 >= 0
- 実車距離 <= 総走行距離
- 内訳合計が総営収と乖離している場合は警告（差異額表示）

**保存後**:
- カレンダーに戻る
- トースト「保存しました」

### 入力補助

- 直近5回の平均値を「目安」として薄く表示
- 数値キーパッドを採用
- フォーカスを次のフィールドに自動移動

## サマリー画面

### レイアウト

- 上部: 月選択（横スワイプで前後月）、月締め期間表示
- カード1: 出番数（実績/予定）、達成率
- カード2: 売上（総額、平均、最高日）
- カード3: 実車率（平均、ベスト日）
- カード4: 拘束時間（合計、上限比、警告）
- カード5: 推定歩合給
- グラフ: 日別営収バーチャート（fl_chart）

### 集計期間

`AppSettings.monthlyClosingDay` に基づく:
- 締め日15日 → 前月16日〜当月15日
- 締め日が月末（31）なら暦月どおり

### エクスポート

右上メニューから「CSV出力」「共有」

CSV項目: 日付, 種別, 総営収, 税抜営収, 現金, キャッシュレス, その他, 総走行, 実車, 回数, 拘束時間, メモ

## 設定画面

### サイクル管理

- 現在のサイクル表示（種別配列をビジュアル表示）
- 「サイクルを編集」「新しいサイクルに切り替え」
- 過去のサイクル履歴一覧

#### サイクル編集

- パターン名
- 勤務形態
- 種別配列（ドラッグ可能なチップ、追加/削除ボタン）
- 起点日
- プレビュー（向こう30日のカレンダーを下に表示し、編集を即時反映）

#### サイクル切替

- 新パターン作成
- 「いつから切替えますか？」で適用開始日選択
- 旧パターンの `validUntil` を自動設定

### 賃金設定

- 足切り額（円）
- 歩率（%）
- 「月の予想歩合給」プレビュー

### 通知設定

- 出番前日リマインド（時刻指定）
- 月末拘束時間集計通知
- 希望休出し忘れ防止（毎月◯日に通知）

### データ管理

- バックアップ（JSON 出力）
- 復元（JSON 取り込み）
- CSV エクスポート（月単位）
- 全データ削除（確認ダイアログ2段階）

### 表示設定

- テーマ（システム/ライト/ダーク）
- 表示名カスタマイズ（明け番→明休 等）

### このアプリについて

- バージョン情報
- プライバシーポリシー（外部リンク）
- 利用規約
- フィードバック（メーラー起動）
- 開発者情報

## 改善基準告示アラート

### 表示場所

- カレンダー画面上部にバッジ（警告/超過）
- サマリー画面のカード4

### 判定タイミング

- アプリ起動時
- 売上/勤務時間記録後
- サイクル変更時
- 月初に通知（任意）

### アラートメッセージ

```
[警告] 今月の拘束時間が上限の90%（236h）を超えました。
[超過] 今月の拘束時間が上限262hを超えました。
[休息不足] 12/15の勤務後、休息時間が18hでした（20h未満）。
[出番数超過] 月の出番が14回に達しました。
```

末尾に「※目安です。正確には会社の労務担当者にご確認ください。」を必ず添える。

## 広告表示

### 配置

- カレンダー画面下部: AdMobバナー
- サマリー画面下部: AdMobバナー
- 設定画面と日詳細画面には**広告を出さない**（操作の邪魔を避ける）

### プレミアム購入後

- 全広告非表示
- 「便利グッズ」アフィリエイト枠は残す（収益化とユーザー利便性の両立）

### アフィリエイト枠

- 設定画面の独立したセクション「お得な情報」
- 4〜6項目の固定リンク（A8.net 経由）
- 外部ブラウザで開く（WebViewは使わない）
- 例: タクシー転職、ドラレコ、ETCカード、確定申告サービス

## エラーハンドリング

- DB エラー: トーストで「保存に失敗しました。再度お試しください」、内部ログに詳細
- 不正な入力: フィールド下に赤字で具体的なエラー
- ネットワーク不要設計のため、オフライン警告は表示しない
