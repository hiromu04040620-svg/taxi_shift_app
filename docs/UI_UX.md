# UI / UX ガイドライン

## デザイン原則

1. **シンプルさ最優先**: 運転中・休憩中の片手操作を想定
2. **大きめのタップ領域**: 最小 44x44 pt（Apple HIG準拠）
3. **コントラスト重視**: 屋外での視認性
4. **数値入力の最適化**: 数値キーパッドを徹底
5. **戻り道を明示**: ナビゲーションの迷子を作らない

## カラーパレット

### ブランドカラー

- Primary: `#1E3A8A`（落ち着いた紺色、夜空・タクシーらしさ）
- Secondary: `#F59E0B`（タクシーの黄、アクセント）
- Background (Light): `#FAFAFA`
- Background (Dark): `#121212`
- Surface (Light): `#FFFFFF`
- Surface (Dark): `#1E1E1E`

### シフト種別カラー

DOMAIN_GLOSSARY.md の表に準ずる。ダークモードでは彩度を落として再定義。

## タイポグラフィ

- 標準フォント: システムフォント（iOS: SF Pro, Android: Roboto）
- 日本語フォント: Noto Sans JP（必要に応じて）
- スケール:
  - Display: 32sp
  - Headline: 24sp
  - Title: 20sp
  - Body: 16sp
  - Caption: 12sp
- ユーザー設定の動的フォントサイズに追従

## レイアウト

- セーフエリア必須
- BottomNavigationBar: カレンダー / サマリー / 設定 の3つ
- 主要画面は AppBar に画面タイトル＋アクションボタン
- パディング: 標準16dp、密集箇所は8dp

## ジェスチャー

- 横スワイプ: 前月/翌月
- 縦スワイプ（カレンダー上）: 月/週表示切替
- 長押し: コンテキストメニュー（クイック編集）
- 引っ張って更新: 採用しない（オフライン前提のため）

## アクセシビリティ

- Semantics ラベルを全アクションに付与
- フォーカス順序を明示
- コントラスト比 WCAG AA 以上
- VoiceOver / TalkBack で操作可能なことを確認

## マイクロインタラクション

- 保存時: 軽いハプティクス＋トースト
- エラー時: 強めのハプティクス
- 画面遷移: Material のデフォルトアニメーション
- 過剰な装飾は避ける

## 空状態・ローディング

- データ無し: イラスト＋短い説明＋アクション（「最初の売上を記録する」等）
- ローディング: スケルトン > Indicatorの順で検討
- 起動直後にDBアクセスがある場合は初回のみスプラッシュで吸収

## アイコン

- Material Icons 標準を採用
- 必要に応じて `flutter_svg` でカスタムSVG
- カラーアイコンは最小限

## トースト・ダイアログ

- 軽い通知: SnackBar（Material）
- 確認: AlertDialog
- 破壊的操作（削除等）: 2段階確認

## 数値入力UX

- 円表記: 3桁カンマ区切りで表示、入力中も自動整形
- 距離: 「123.4 km」形式
- 入力フォーカス時に既存値を全選択
- 「次へ」ボタンで次フィールドに自動移動
